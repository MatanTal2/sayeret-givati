## Per-task verification checklist

Run these on every Phase 1–3 branch before opening a PR:

- `npm run lint` passes.
- `npm test` passes; new behavior covered by Jest tests under `src/lib/__tests__/` or `src/components/<domain>/__tests__/`.
- `npm run build` succeeds.
- For UI tasks: start `npm run dev`, exercise the feature in browser (golden path + 1 edge case), verify no console errors.
- For Firestore-touching tasks: confirm reads/writes go through `src/lib/` services per CLAUDE.md (no raw `getDoc`/`setDoc` in components or hooks).
- Update docs under `docs/codebase/src/` and (where applicable) `docs/firebase-operations.md`, `docs/duplications.md`, `docs/spec/` per the project's docs-with-code rule.
- PR description must explicitly state any out-of-band manual steps (e.g. "deploy storage rules", "deploy firestore rules", "regenerate service account").
